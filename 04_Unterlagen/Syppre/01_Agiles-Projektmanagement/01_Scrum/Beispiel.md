# 📋 Product Backlog

## 🧩 Epic: Aufgabenverwaltung

### 👤 User Stories

1. Als Benutzer möchte ich Aufgaben hinzufügen, damit ich sie speichern kann  
2. Als Benutzer möchte ich Aufgaben anzeigen, damit ich sie sehen kann  
3. Als Benutzer möchte ich Aufgaben löschen, damit meine Liste übersichtlich bleibt

# 🔄 Erklärung

👉 Jeder Sprint bearbeitet gezielt **User Stories**

👉 Ein Sprint benötigt nicht zwingend einen Namen 

👉 Tasks sind immer:
- konkret
- klein
- einer User Story zugeordnet

---

# 🧠 Warum ist das wichtig?

👉 Dadurch weiß man immer:
- warum man etwas baut
- welchen Nutzen es hat
- wann eine Funktion fertig ist
